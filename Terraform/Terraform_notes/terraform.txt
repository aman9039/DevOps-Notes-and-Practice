How Terraform Works:
===================

Remember only this picture:

Write Code
   ↓
terraform init
   ↓
terraform plan
   ↓
terraform apply
   ↓
Infrastructure Created

Only three commands.

Step 1: terraform init
======

Imagine:
Installing Terraform plugins.

Ex.
If using AWS, Terraform downloads AWS plugin. Think: init = setup

Step 2: terraform plan
======

Terraform asks:
"Boss, this is what I am going to create."

Example:
+ create server
+ create database
+ create load balancer

Nothing is created yet.

Think:
plan = preview

Step 3: terraform apply
======

Terraform says:
"Okay, now I will create everything."

Think:
apply = execute

Bonus Command: terraform destroy
=============
Deletes everything.

Think:
destroy = nuclear button

Very useful in testing environments.

Q.What is Terraform?
Answer: Terraform is an Infrastructure as Code tool developed by HashiCorp. It is used to create, manage,and provision infrastructure using code instead of manually creating resources from the cloud console.

INIT → PLAN → APPLY → DESTROY
INIT (Buy tools) --> PLAN (Create blueprint) --> APPLY (Build house) --> DESTROY (Demolish house)

Terraform Core Concepts
=======================

There are only a few concepts.
Everything else is built on top of them.

1. Provider

Question:
Which cloud should Terraform talk to?
Examples: AWS,Azure,GCP,Kubernetes,VMware

Terraform --> Translator ---> AWS API
** Provider = Cloud Translator

2. Resource

Question:
What should Terraform create?

Examples:
EC2,VPC,Security Group,S3 Bucket,EKS Cluster
** Resource = Something to create.

3. Arguments
Resources need details.

Example:

Server Size?
Operating System?
Subnet?
Security Group?

These details are arguments.

Example:
instance_type = "t2.micro"
** Arguments = Configuration details.
